
Simple leak detector
* detect memory leaks with malloc et al.
* show trace to the leak source (option)
* check if memory is deallocated in different module than allocated
* expose interface to hook in which memory belongs to smart pointer
* analyse data to see if there are cycles in the smart pointers and report those
* simple analysis of allocation source and allocation patterns
  * for example combined allocations (object + shared_ptr control block), alloc+dealloc = could be move?
* as little impact on performance as possible

Technical:
* Shared object preload on linux
* dll to link against on windows (provide header with lib pragmas for msvc)
* first output in plain text only
* hook interface:
  * add_pointer_owner(const void* owner_address, const void* containee_address, std::uint32_t size);
    * if containee is allocated using our stuff, we have the size, what if not? Try to provide the size here?
  * remove_pointer_owner(const void* owner_address, const void* containee_address);
  * move_pointer_owner(const void* old_owner, const void* new_owner, const void* containee_address);
